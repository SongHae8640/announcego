# 알리다고(AnnounceGo) 광고 관리 시스템 요구사항 정의서

## 1. 프로젝트 개요

### 1.1 프로젝트 목적
- 현재 120여개 이상의 병원 광고를 카카오톡 단톡방으로 관리하고 있으나, 관리의 복잡성으로 인한 효율성 저하
- 여러 광고 업체의 서비스를 한 곳에서 통합 관리할 수 있는 시스템 필요
- 병원이 자신의 광고 상황(순위, 노출 등)을 실시간으로 확인할 수 있는 플랫폼 제공

### 1.2 비즈니스 배경
- 한의원 광고 관리의 차별화 포인트로 활용
- 투명한 광고 성과 공유를 통한 신뢰 구축

---

## 2. 시스템 구성

### 2.1 기본 정보
- **서비스명**: 알리다고(AnnounceGo) 어드민 사이트
- **플랫폼**: PC 웹사이트 중심 (모바일은 추후 개발)
- **공개 범위**: 비공개 시스템 (검색엔진 노출 차단)
- **예상 데이터량**: 하루 최대 600건 이상 (120개 업체 × 약 5건)

### 2.2 사용자 역할

#### 2.2.1 관리자 (알리다고)
- 회원가입 승인 관리
- 병원 계정 목록 조회 및 광고업체 계정 목록 조회
- 모든 광고업체 및 병원의 1:1 문의 내역 확인 가능

#### 2.2.2 광고업체 (120개)
- 회원가입 (담당 병원명 입력, 마케팅 타입 선택)
- 담당 병원의 마케팅 타입별 데이터 입력
- 관리자와 1:1 문의/소통

#### 2.2.3 병원
- 회원가입
- 자신의 광고 상황(순위, 노출 등) 조회
- 관리자와 1:1 문의/소통

### 2.3 업체-병원 관계
- **N:1 매칭 구조**: 하나의 병원에 여러 광고업체가 담당할 수 있음
- 광고업체가 회원가입 시 담당 병원명을 입력

## 3. 주요 기능 요구사항 (역할별)

### 3.1 관리자 기능

#### 3.1.1 병원 계정 목록 조회
- 병원 계정 정보 조회 (병원명, 원장명, 연락처, 가입일 등)
- 병원별 담당 광고업체 목록 조회
- 병원 계정 상태 관리 (활성/비활성)

#### 3.1.2 광고업체 계정 목록 조회
- 광고업체 계정 정보 조회 (업체명, 담당자, 연락처, 담당 병원명, 마케팅 타입 등)
- 광고업체 계정 상태 관리 (활성/비활성)

#### 3.1.3 회원가입 승인 관리
- 병원 회원가입 요청 목록 조회
- 광고업체 회원가입 요청 목록 조회
- 가입 정보 검토 및 승인/반려 처리
- 승인/반려 사유 입력

#### 3.1.4 전체 데이터 조회
- 모든 광고업체의 마케팅 타입별 데이터 조회
  - 블로그 상위 노출: 회차별 순위 데이터
  - 브랜드 블로그: 키워드-URL 매핑 현황
  - 카페/영수증 블로그: 캘린더 일정 및 계약 정보
- 모든 병원의 광고 현황 조회
- 통계 및 리포트 기능

#### 3.1.5 문의 관리
- 모든 광고업체 및 병원의 1:1 문의 내역 확인
- 문의에 대한 답변 작성

### 3.2 광고업체 기능

#### 3.2.1 회원가입
- 계정 정보 입력 (ID, 비밀번호)
- 업체 정보 입력 (업체명, 담당자, 연락처)
- 담당 병원명 입력
- 마케팅 타입 선택 (복수 선택 가능)

#### 3.2.2 마케팅 타입별 데이터 입력

**블로그 상위 노출:**
- 회차 정보 입력 (회차, 키워드, 결제일, 노출 형태, 노출 시작일)
- 노출 형태: 건바이건, 월보장 (복수 선택 가능)
- 일별 순위 데이터 입력 (노출 시작일부터 25일간)
- 키워드별 순위 입력 지원
- 모든 키워드 25번 입력 시 회차 종료

**브랜드 블로그:**
- 키워드, URL, 순위, 반영(On/Off) 입력

**카페:**
- 캘린더 형태의 일정 관리
- 일자별 내용 작성/수정
- 계약 정보 입력 (계약 내용, 계약일, 비고)

**영수증 블로그:**
- 캘린더 형태의 일정 관리
- 일자별 내용 작성/수정
- 계약 정보 입력 (계약 내용, 계약일, 비고)

#### 3.2.3 데이터 조회
- 담당 병원의 입력한 데이터 조회
- 데이터 입력 내역 히스토리

#### 3.2.4 1:1 문의
- 관리자에게 문의 작성
- 본인 작성 문의 내역 조회
- 관리자 답변 확인

### 3.3 병원 기능

#### 3.3.1 회원가입
- 계정 정보 입력 (ID, 비밀번호)
- 병원 정보 입력 (병원명, 원장명, 원장 연락처)

#### 3.3.2 광고 현황 조회
- 자신의 병원을 담당하는 업체들의 마케팅 타입별 데이터 조회
  - 블로그 상위 노출: 회차별 순위 현황
  - 브랜드 블로그: 키워드별 URL 및 순위
  - 카페/영수증 블로그: 캘린더 일정 확인
- 광고 성과 그래프/차트

#### 3.3.3 1:1 문의
- 관리자에게 문의 작성
- 본인 작성 문의 내역 조회
- 관리자 답변 확인

## 5. 기술적 고려사항

### 5.1 성능
- 하루 600건 이상의 데이터 처리 안정성 확보
- 동시 접속자 수 대비 응답 속도 최적화

- 병원 테이블 관리
- 마케팅 타입별 데이터 테이블 분리
  - 블로그 상위 노출: 회차 관리 및 일별 순위 데이터
  - 브랜드 블로그: 키워드-URL 매핑 데이터
  - 카페/영수증 블로그: 캘린더 일정 데이터
