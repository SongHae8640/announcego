# 알리다고(AnnounceGo) 광고 관리 시스템 요구사항 정의서

## 1. 프로젝트 개요

### 1.1 프로젝트 목적
- 현재 120여개 이상의 병원 광고를 카카오톡 단톡방으로 관리하고 있으나, 관리의 복잡성으로 인한 효율성 저하
- 여러 광고 업체의 서비스를 한 곳에서 통합 관리할 수 있는 시스템 필요
- 병원이 자신의 광고 상황(순위, 노출 등)을 실시간으로 확인할 수 있는 플랫폼 제공

### 1.2 비즈니스 배경
- 한의원 광고 관리의 차별화 포인트로 활용
- 투명한 광고 성과 공유를 통한 신뢰 구축

---

## 2. 시스템 구성

### 2.1 기본 정보
- **서비스명**: 알리다고(AnnounceGo) 어드민 사이트
- **플랫폼**: PC 웹사이트 중심 (모바일은 추후 개발)
- **공개 범위**: 비공개 시스템 (검색엔진 노출 차단)
- **예상 데이터량**: 하루 최대 600건 이상 (120개 업체 × 약 5건)

### 2.2 사용자 역할

#### 2.2.1 관리자 (알리다고)
- 병원 목록 관리 (등록, 수정, 삭제)
- 광고업체 계정 생성 및 관리 (담당 병원 지정)
- 모든 광고업체 및 병원 정보 조회/관리
- 계약 체결 관리 (미결제/결제완료 수정 권한)
- 모든 1:1 문의 내역 확인 가능

#### 2.2.2 광고업체 (120개)
- 담당 병원의 광고 순위 데이터 입력
- 자사의 계약 체결 정보 입력 및 수정
- 광고 계약 만료일 D-day 확인
- 관리자와 1:1 문의/소통

#### 2.2.3 병원
- 로그인 없이 사용하는 사용자
- 자신의 광고 상황(순위, 노출 등) **조회 전용**

### 2.3 업체-병원 관계
- **N:1 매칭 구조**: 하나의 병원에 여러 광고업체가 담당할 수 있음
- 관리자가 광고업체 추가 시 담당 병원을 지정

## 3. 주요 기능 요구사항 (역할별)

### 3.1 관리자 기능

#### 3.1.1 병원 목록 관리
- 병원 등록 (병원명, 연락처, 주소 등)
- 병원 정보 수정
- 병원 삭제 (담당 광고업체가 없는 경우에만)
- 병원별 담당 광고업체 목록 조회

#### 3.1.2 광고업체 계정 관리
- 광고업체 계정 생성 및 고유 ID 부여
- **담당 병원 지정** (병원 목록에서 선택)
- 필수 정보 항목 (변경 가능):
  - 업체명
  - 담당 병원 (선택)
  - 연락처
  - 이메일
  - 기타 관리 정보

#### 3.1.3 전체 데이터 조회
- 모든 광고업체의 순위 데이터 조회
- 모든 병원의 광고 현황 조회
- 통계 및 리포트 기능

#### 3.1.4 계약 체결 관리
- 모든 광고업체의 계약 체결 정보 조회
- 계약 체결 상태 변경 (미결제 ↔ 결제완료)
- 계약 체결 이력 관리

#### 3.1.5 문의 관리
- 모든 광고업체의 1:1 문의 내역 확인
- 문의에 대한 답변 작성

### 3.2 광고업체 기능

#### 3.2.1 광고 데이터 입력
담당 병원의 광고 순위 데이터 입력:
- 키워드
- 순위
- 날짜
- 기타 광고 성과 지표

#### 3.2.2 데이터 조회
- 담당 병원의 입력한 데이터 조회
- 데이터 입력 내역 히스토리

#### 3.2.3 계약 체결 관리
자사의 계약 체결 정보 입력 및 수정:
- 은행명
- 계좌번호
- 입금주
- 결제금액
- 결제일
- 계약만료일

#### 3.2.4 계약 만료일 알림
- 대시보드에 **계약만료일 D-day 표시**
- 만료 임박 시 시각적 알림 (예: D-7, D-3, D-day)

#### 3.2.5 1:1 문의
- 관리자에게 문의 작성
- 본인 작성 문의 내역 조회
- 관리자 답변 확인

### 3.3 병원 기능

#### 3.3.1 광고 현황 조회 (조회 전용)
- 자신의 광고 순위 확인
- 키워드별 노출 현황 확인
- 광고 성과 그래프/차트

- ✅ **결정사항**: 관리자가 계정정보(ID/초기 비밀번호)를 부여하고, 광고업체는 최초 로그인 시 비밀번호 변경창에서 비밀번호를 변경해야 대시보드로 이동 가능

- [ ] 연락처

**Q3. 병원의 광고 현황 조회 방식**
- ✅ **결정사항**: 링크 + 간단한 인증 (병원 전화번호 뒷자리 입력)

**Q4. "계약 체결"의 정확한 의미 및 프로세스**
- ✅ **결정사항**:
  - 결제 연동 기능 없음 (PG 연동 없이 수동 관리)
  - 알리다고와 광고업체 간 계약 체결 후 시스템 사용 시작
  - **광고업체 입력 항목**: 은행명, 계좌번호, 입금주, 결제금액, 결제일, 계약만료일
  - **알리다고 확인 작업**: 실제 입금 확인 후 상태를 "결제완료"로 변경
  - **역할 분담**: 입력은 광고업체, 최종 확인 및 상태 변경은 알리다고

**Q5. 1:1 문의 기능의 UI/UX 방식**
- ✅ **결정사항**: 전통적인 게시판 QnA 방식 (제목, 내용, 답변)
